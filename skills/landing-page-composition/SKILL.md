# Landing Page Composition

Shape a landing page into a strong narrative, not a bag of sections. This skill is for product pages, official sites, launch pages, and feature landing flows.

## When to use
- Build or refine a homepage, official site, or feature landing page
- Improve hero sections, section order, narrative rhythm, or CTA strategy
- Turn product capabilities into a page that communicates value and momentum
- Tighten a page that already has content but no convincing flow

## Workflow
1. Define the page goal: awareness, conversion, qualification, trust-building, or launch momentum.
2. Identify the audience and what they need to believe by the end of the scroll.
3. Choose a section arc using `references/structure.md`.
4. Design the hero first: promise, evidence, primary CTA, secondary CTA, and supporting visual.
5. Sequence the remaining sections so each one earns the next.
6. Remove redundant sections and repeated claims.
7. If implementing in code, make the layout read well on both desktop and mobile before polishing ornament.

## Composition rules
- Every section should have a job.
- Prefer fewer, stronger sections over many shallow ones.
- The page should alternate between tension and relief: dense proof, then clean breathing room.
- Keep CTA strategy consistent. Do not rotate the main ask every screen.
- If the product is sophisticated, explain it progressively instead of front-loading jargon.

## Output shape
- Page goal
- Proposed narrative arc
- Hero recommendation
- Section order with one-line purpose each
- Notes on proof, CTA, and visual pacing
