# Landing Page Structure Patterns

## Strong Default Arc
1. Hero: promise + proof hint + CTA
2. Why this matters now: problem framing or shift in how work is done
3. Product mechanism: how it works in plain language
4. Key capabilities: grouped into 2 to 4 clear themes
5. Proof: examples, screenshots, results, trust signals
6. Workflow or team fit: how it works in real use
7. Final CTA: one decisive ask

## Hero Checklist
- One sharp promise, not three competing claims
- One primary CTA
- Optional secondary CTA only if it serves a different intent
- Supporting line should clarify, not repeat
- Visual should either explain the product or intensify the mood, not both badly

## Section Pacing
- Do not stack three dense text sections in a row
- Use screenshots or diagrams where mechanism is hard to picture
- Collapse repeated benefit statements into one stronger section

## Official Site Guidance
- Homepages should explain the product fast and then deepen trust
- Feature pages can go more technical after the first proof beat
- If the product is new, narrative clarity matters more than exhaustive coverage
